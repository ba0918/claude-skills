## 📝 What & Why

brainstorm スキルの発散フェーズにチーム議論を導入する新スキル `team-brainstorm` を作成する。1人の Claude では視点が偏りがちな発散フェーズを、4つの異なる思考スタイル（Challenger/Explorer/Connector/Grounded）で多角化し、アイデアの質と多様性を劇的に向上させる。

## 🎯 Goals

- 独立スキル `team-brainstorm` を新規作成する
- 発散向き4ロール（Challenger/Explorer/Connector/Grounded）+ optional Domain Expert を定義する
- 独立発散 → 論争メモリ分類（Accepted/Controversial/Frontier）→ ユーザーフィードバックの三段階フローを実装する
- ユーザーが議論に積極参加できるインタラクティブなセッション設計にする
- 既存 brainstorm スキルとの共存（session/wrap/plan/list/resume の5モード対応）

## 📐 Design

### Step 1: ロール定義ファイルの作成

#### Files to Change

```
skills/team-brainstorm/references/brainstorm-roles.md - 新規作成: 4ロール + optional Domain Expert の定義
```

#### Key Points

- **Challenger（挑戦者）**: Security の脅威分析能力を背景に、「このアイデアの穴は？どう壊れる？」を追求
- **Explorer（探索者）**: Architect の構造思考を背景に、「制約を外したら何が見える？統合するなら？」を追求
- **Connector（接続者）**: Performance のシステム思考を背景に、「この案とあの案を組み合わせたら？類似パターンは？」を追求
- **Grounded（地に足）**: Pragmatist の実装知見を背景に、「で、最初の一歩は何になる？」を追求
- **Domain Expert（optional）**: ドメイン固有の制約・規制を提供。ユーザーが起動時に選択
- 各ロール定義に「背景知識（team-config.md への参照）」「ブレスト時の視点」「典型的な質問」「出力形式」を含める
- ロール名は発散向きの動詞的ネーミング（心理的モード切り替え効果）

### Step 2: 論争メモリのテンプレート作成

#### Files to Change

```
skills/team-brainstorm/references/session-template.md - 新規作成: Session 記録テンプレート（論争メモリ含む）
```

#### Key Points

- **論争メモリの3分類**:
  - **Accepted**: 全員が「面白い」と認めた案
  - **Controversial**: ロール間で意見が分かれている案（対立構造を明記）
  - **Frontier**: 実装方法は不明だが革新的な可能性がある案
- 各ラウンドごとに論争メモリを更新・蓄積する構造
- ラウンド N+1 では Controversial と Frontier に集中して深掘り
- Wrap 時に idea-template.md（既存 brainstorm と同じフォーマット）に fold する

### Step 3: 発散フローの定義

#### Files to Change

```
skills/team-brainstorm/references/brainstorm-flow.md - 新規作成: 三段階発散フロー
```

#### Key Points

- **三段階フロー**:
  1. **開放的発散（Phase 1）**: 各ロール独立で案を3-5個列挙（並行実行、相互参照禁止）
  2. **論争メモリ分類（Phase 2）**: Lead がチーム全案を Accepted/Controversial/Frontier に分類
  3. **ユーザーフィードバック + 深掘り（Phase 3）**: ユーザーが質問・反応 → チームが深掘り
- **ラウンド制**: 上記三段階を1ラウンドとして、ユーザーが「もっと掘りたい」なら次ラウンドへ
- **各ラウンド終了時に進捗サマリーを表示**:
  ```
  ── Round {N} ──
  Accepted: {count}件
  Controversial: {count}件（{対立の概要}）
  Frontier: {count}件
  ```
- **早期収束条件**: ユーザーが「wrap」「まとめて」「終わり」と言ったら終了
- **合意形成を目指さない**: 矛盾・対立を可視化して記録する（team-plan/team-cycle との根本的な違い）

### Step 4: SKILL.md の作成

#### Files to Change

```
skills/team-brainstorm/SKILL.md - 新規作成: メインロジック
```

#### Key Points

- **5つのワークフロー**: session / wrap / plan / list / resume（既存 brainstorm と同構造）
- **Session ワークフロー**:
  1. テーマ取得（$ARGUMENTS または AskUserQuestion）
  2. Domain Expert 参加の有無を確認（optional）
  3. TeamCreate → 4ロール spawn（Explore サブエージェント型）
  4. キックオフ: テーマ + ドメイン制約をチームに共有
  5. 発散ループ（brainstorm-flow.md 準拠）
  6. TeamDelete（try-finally パターンで保証）
  7. Wrap への誘導
- **Edit/Write 禁止ルール**: 既存 brainstorm と同じ。Session/Resume 中はファイル編集禁止
- **Wrap ワークフロー**: 論争メモリから idea-template.md を生成し `docs/ideas/` に保存
- **Plan ワークフロー**: Skill ツールで `claude-skills:plan-create`（または `team-plan`）を呼び出す
- **frontmatter の description**: 「チーム議論型のブレインストーミング。4つの異なる思考スタイル（Challenger/Explorer/Connector/Grounded）で多角的にアイデアを発散させる」

### Step 5: コマンドファイルの作成

#### Files to Change

```
commands/team-brainstorm.md - 新規作成: /team-brainstorm コマンド
commands/team-brainstorm-wrap.md - 新規作成: /team-brainstorm-wrap コマンド
```

#### Key Points

- コマンドは Skill ツール経由でスキルを呼び出す薄いラッパー
- `team-brainstorm` → `skills/team-brainstorm/SKILL.md` の session ワークフロー
- `team-brainstorm-wrap` → `skills/team-brainstorm/SKILL.md` の wrap ワークフロー
- list / plan / resume は既存 brainstorm のコマンドを共用（idea-status.md が共通なので）
