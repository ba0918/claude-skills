# doc-write — LLMとのやり取りをリーダブルなドキュメントに昇華するスキル

**Created:** 2026-03-19
**Status:** 💡 Idea
**Tags:** `skill`, `documentation`, `workflow`

---

## Summary

セッション中の議論・調査結果・Web調査などを、図付きの構造化ドキュメントとして docs/ に出力するスキル。想定読者に応じた粒度調整、テンプレート自動選択、フィードバックループによる品質向上を特徴とする。

## Key Discussion Points

- **既存スキルとの住み分け**: brainstorm-wrap（軽量メモ）、investigate（調査レポート）とは異なり、「知見のリーダブルなドキュメント化」に特化。investigateの出力を入力にする関係も想定
- **入力ソース3パターン**: (1) 現在のコンテキスト（今の会話をまとめて）、(2) 既存ファイル（過去のメモ・レポート統合）、(3) テーマ指定でWeb調査してドキュメント化
- **Mermaid図の品質問題**: LLM生成のMermaid図は配置がめちゃくちゃ・縮小されすぎで読みづらい。ノード数制限・分割戦略・ラベル短縮等のガイドラインをreferencesに持たせるが、実際の効果は使いながら調整
- **想定読者の確認**: 自分/チーム/外部発表で粒度が変わるため、起動時に必ず確認する（唯一の必須質問）
- **イテレーション**: 初稿生成後にフィードバック→修正ループ。brainstorm-resume的に後から再編集も可能（doc-write-resume）
- **Web調査の深さ**: 文脈判断。「調べて」→ライト、「徹底的に調べて」→ディープ。最終的にユーザーと壁打ちするので往復回数が変わるだけ

## Decisions & Conclusions

- **スキル名**: `doc-write`（doc-checkとの紛らわしさを避けつつシンプル）
- **テンプレート**: テックノート / ADR / ディスカッションサマリーの3型、内容に応じて自動選択
- **初回質問フロー**: 想定読者のみ必須で確認。テーマは引数/コンテキストから推測、テンプレートは自動選択がデフォルト
- **出力先**: `docs/` 配下の適切なディレクトリ
- **Mermaidガイドライン**: 最低限のルールで開始し、使いながら育てる方針
- **skill-creator使用**: スキル作成時はskill-creatorを使って効率化する
