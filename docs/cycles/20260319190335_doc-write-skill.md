# doc-write スキル作成

## What & Why

LLMとのやり取り・調査結果・Web調査などを、Mermaid図付きのリーダブルなドキュメントとして `docs/` に出力するスキルを作成する。

既存の brainstorm-wrap（軽量メモ）や investigate（調査レポート）とは異なり、「読者を意識した構造化ドキュメント」に特化する。入力ソースは現在のコンテキスト / 既存ファイル / Web調査の3パターン。フィードバックループで品質を磨き、後から再編集も可能。

---

## Steps

### Step 1: skill-creator でスキルの雛形を作成

**What:** skill-creator スキルを使って `doc-write` スキルのディレクトリ構造と SKILL.md を生成する。

**How:**
- Skill ツールで `skill-creator` を起動
- 以下の要件を伝える:
  - スキル名: `doc-write`
  - 概要: LLMとのやり取り・調査結果・Web調査をリーダブルなドキュメントに昇華するスキル
  - ワークフロー: write（メイン） / resume（再編集）の2ワークフロー
  - 想定読者の確認（必須質問）
  - 入力ソース3パターン（コンテキスト / ファイル / Web調査）
  - テンプレート自動選択（テックノート / ADR / ディスカッションサマリー）
  - Mermaid図生成ガイドライン
  - フィードバック→修正ループ
  - 出力先: `docs/writings/{slug}.md`（slug = `YYYY-MM-DD_{kebab-title}`）
  - エラーハンドリング: Web調査失敗時はユーザーに通知して手動入力に切替、空入力時は AskUserQuestion で再入力要求、テンプレート選択曖昧時はユーザーに確認
  - resume ワークフロー: ファイルパス or slug 指定で対象を特定。docs/writings/ から一覧表示して選択も可。保持状態はドキュメントファイル自体（メタデータを frontmatter に埋め込み: 想定読者、テンプレート型、最終更新日）

**Files:**
- `skills/doc-write/SKILL.md` (create)
- `skills/doc-write/references/` (create — テンプレート・ガイドライン)

**Verification:**
- [x] SKILL.md が生成されている
- [x] references/ にテンプレートファイルが配置されている
- [x] ワークフロー分岐（write / resume）が定義されている

### Step 2: SKILL.md のレビューと調整

**What:** skill-creator が生成した SKILL.md を壁打ちの決定事項と照合し、不足・過剰がないか確認・修正する。

**How:**
- 壁打ちの Decisions & Conclusions と照合:
  - 想定読者の確認が必須質問として入っているか
  - 入力ソース3パターンが網羅されているか
  - Web調査の深さが文脈判断になっているか（「調べて」→ライト / 「徹底的に」→ディープ）
  - テンプレート自動選択の仕組みが入っているか
  - フィードバックループの仕組みが入っているか
  - Mermaid図ガイドラインが references に分離されているか
  - resume ワークフローが具体的に定義されているか（対象ファイル特定方法、frontmatter メタデータ、一覧選択UI）
  - エラーハンドリングが定義されているか（Web調査失敗、空入力、resume対象未存在、テンプレート選択曖昧時）
  - 出力先が `docs/writings/{slug}.md` になっているか
- 必要に応じて Edit で修正

**Files:**
- `skills/doc-write/SKILL.md` (edit)
- `skills/doc-write/references/*.md` (edit)

**Verification:**
- [x] 壁打ちの全決定事項が SKILL.md に反映されている
- [x] Mermaid ガイドラインに最低限のルール（ノード数制限・分割戦略・ラベル短縮）が含まれている

### Step 3: コマンドファイルの作成

**What:** `/doc-write` と `/doc-write-resume` のコマンドファイルを作成する。

**How:**
- 既存コマンド（brainstorm.md, brainstorm-resume.md）をパターンとして使用
- `commands/doc-write.md`: Skill ツールで `doc-write` を実行する薄いラッパー
- `commands/doc-write-resume.md`: Skill ツールで `doc-write` を `resume $ARGUMENTS` で実行

**Files:**
- `commands/doc-write.md` (create)
- `commands/doc-write-resume.md` (create)

**Verification:**
- [x] frontmatter の description が適切に設定されている
- [x] Skill ツール呼び出しのフォーマットが既存コマンドと統一されている

### Step 4: CLAUDE.md の更新

**What:** コマンド→スキルのマッピング表とスキル一覧を更新する。

**How:**
- CLAUDE.md のコマンド→スキル関係テーブルに追加:
  - `commands/doc-write.md → skills/doc-write/SKILL.md`
  - `commands/doc-write-resume.md → skills/doc-write/SKILL.md (resume ワークフロー)`
- 主要スキルテーブルに追加:
  - `doc-write` | LLMとのやり取り・調査結果をリーダブルなドキュメントに昇華。Mermaid図付き

**Files:**
- `CLAUDE.md` (edit)

**Verification:**

- [x] スキル一覧テーブルが更新されている

### Step 5: 動作確認

**What:** 実際にスキルを起動して基本フローが動くことを確認する。

**How:**
- `install.sh` を実行してシンボリックリンクを作成（install.sh は glob で自動検出するため編集不要）
- 以下を確認:
  - `~/.claude/skills/doc-write` シンボリックリンクが正しく作成されている
  - `~/.claude/commands/doc-write.md` シンボリックリンクが正しく作成されている
  - SKILL.md 内の references パスが正しく解決できる（相対パスで参照可能か）
  - コマンドの frontmatter が正しくパースされる

**Verification:**

- [x] シンボリックリンクが正しく作成されている
- [x] SKILL.md 内の references パスが正しく解決できる
- [x] コマンドの frontmatter が正しくパースされる

## Notes

- Mermaid ガイドラインは最低限で開始し、使いながら育てる方針
- テンプレートは3型（テックノート / ADR / ディスカッションサマリー）で開始、必要に応じて追加
- Step 1 で skill-creator を活用することで、スキルの構造的な品質を担保する
