# plan ファイルパス逸脱防止（二重防御）

**Cycle ID:** `20260316021653`
**Started:** 2026-03-16 02:16:53
**Status:** 🟡 Planning

---

## 📝 What & Why

`issue-cycle` 経由で plan を作成した際、LLM が `docs/cycles/` ではなく `docs/plans/` に計画ファイルを作ってしまう問題が発生した。Agent 委譲で Skill を呼び出す際に、LLM がスキルの指示を正しく読まずにパスを推測するのが原因。予防（指示強化）+ 検知（パス検証）の二重防御で対策する。

## 🎯 Goals

- issue-cycle の SKILL.md で plan 出力先パスを明示的に指示する（予防レイヤー1）
- cycle コマンドの Phase 0 で plan ファイルが `docs/cycles/` 配下にあるか検証し、逸脱していたらエラーにする（検知）
- どの経路から cycle が呼ばれても防御が効くようにする
- plan-create（plan/SKILL.md）自体にも出力パスの制約を明記し、根本原因を解消する（予防レイヤー2）

## 📐 Design

### Files to Change

```
skills/issue/SKILL.md       - 更新: Cycle Workflow Step 4 にパス制約を明記
commands/cycle.md            - 更新: Phase 0 にパス検証ステップを追加
skills/plan/SKILL.md         - 更新: Phase 3 に出力パス制約の注意書きを追加
```

### Key Points

- **予防レイヤー1（issue/SKILL.md）**: Cycle Workflow の Step 4 で「plan は必ず `docs/cycles/{timestamp}_{slug}.md` に作成すること」を明示。Agent 経由で伝搬しやすいよう、制約をコメントではなく指示文として記述する
- **予防レイヤー2（plan/SKILL.md）**: Phase 3 に「出力先は必ず `docs/cycles/` 配下であること」の制約を明記する。これが根本対策であり、どの経路から `plan-create` が呼ばれても一貫してパスが正しくなる
- **検知（cycle.md）**: Phase 0 の計画ファイル特定後に、パスが `docs/cycles/*.md` パターンに一致するか検証。一致しない場合はエラーメッセージとともにサイクルを中断する
- **既存フローへの影響なし**: 正しいパスに plan がある場合は何も変わらない。ガードは逸脱時のみ発動

### 具体的な変更内容

#### 1. skills/issue/SKILL.md — Cycle Workflow Step 4 強化

現在:
```
4. Execute `plan-create` via the Skill tool based on the issue content (title and summary)
   - Arguments: Pass the issue's title and summary
```

変更後:
```
4. Execute `plan-create` via the Skill tool based on the issue content (title and summary)
   - Arguments: Pass the issue's title and summary
   - **CRITICAL**: The plan file MUST be created at `docs/cycles/{timestamp}_{slug}.md`. Do NOT use `docs/plans/` or any other directory. Verify the file was created in `docs/cycles/` before proceeding.
```

#### 2. skills/plan/SKILL.md — Phase 3 にパス制約を明記

Phase 3 の冒頭（`**File path:** \`docs/cycles/{timestamp}_{feature-slug}.md\`` の直後）に制約を追加:

```
**CRITICAL**: Plan files MUST be created under `docs/cycles/` directory. Do NOT use `docs/plans/` or any other directory. This constraint applies regardless of how this skill is invoked (directly, via issue-cycle, or any other caller).
```

#### 3. commands/cycle.md — Phase 0 にパス検証追加

Phase 0 の Step 1（計画ファイル特定）の直後に検証ステップを挿入:

```
1.5. パスを検証する
   - 計画ファイルのパスが `docs/cycles/` 配下であることを確認する
   - パスが `docs/cycles/*.md` に一致しない場合:
     ```
     ⛔ CYCLE ABORTED: Plan file is not in docs/cycles/
     Found: {actual_path}
     Expected: docs/cycles/*.md

     Plan files must be located in docs/cycles/.
     If the file was created in the wrong location, move it first:
       mv {actual_path} docs/cycles/
     ```
     サイクルを中断する
```
