# Brainstorm Discussion Flow

チームブレインストーミングの三段階発散フローの詳細手順。

## 概要

1ラウンドは3つのフェーズで構成される。ユーザーが「もっと掘りたい」と言えば次ラウンドへ進む。

```
Phase 1: 開放的発散（各ロール独立、並行実行、相互参照禁止）
  ↓
Phase 2: 論争メモリ分類（Lead が Accepted / Controversial / Frontier に分類）
  ↓
Phase 3: ユーザーフィードバック + 深掘り（ユーザーが質問・反応 → チームが深掘り）
  ↓
ラウンドサマリー表示
  ↓
ユーザー判断: 「もっと掘りたい」→ 次ラウンド / 「wrap」→ 終了
```

## Phase 1: 開放的発散

### 目的

各ロールが独立にアイデアを3-5個生成する。相互参照を禁止し、視点の多様性を最大化する。

### 手順

1. Lead が各ロールに send_message でテーマとコンテキストを共有する
2. 各ロールが **並行で** 独立にアイデアを生成する
3. 各ロールがアイデアを出力し、Lead が wait_agent で受け取る

### 重要なルール

- **相互参照禁止**: Phase 1 では他のロールのアイデアを参照しない
- **批判禁止**: Phase 1 ではアイデアの評価・批判を行わない
- **量を優先**: 質より量。最低3個、できれば5個のアイデアを出す
- **自由度最大**: 実現可能性や妥当性は Phase 3 で議論する

### send_message テンプレート（Phase 1 開始、全員宛）

```
## ブレスト Round {N}: Phase 1 — 開放的発散

### テーマ
{theme}

### コンテキスト
{context}

### 指示
あなたの専門視点から、テーマに関するアイデアを3-5個出してください。
- 他のメンバーの案は参照しないでください（独立発散）
- 批判・評価は不要です（Phase 3 で議論します）
- 量を優先してください
- 自由に発想してください

{round_2_plus_additional_context}
```

Round 2 以降は、前ラウンドの Controversial と Frontier を追加コンテキストとして含める:

```
### 前ラウンドの未解決論点（深掘り対象）

**Controversial:**
{controversial_items}

**Frontier:**
{frontier_items}

上記の論点を深掘りしつつ、新しいアイデアも歓迎します。
```

## Phase 2: 論争メモリ分類

### 目的

全ロールのアイデアを Lead が Accepted / Controversial / Frontier に分類する。

### 手順

1. Lead が全ロールからの報告を wait_agent で受け取る
2. 全アイデアを以下の基準で分類する:
   - **Accepted**: 複数ロールが類似のアイデアを出した、または明らかに有望な案
   - **Controversial**: ロール間で評価が分かれそうな案（Challenger が穴を見つけそう、Grounded が「無理」と言いそう、等）
   - **Frontier**: 実装方法が不明だが、革新的な可能性を持つ案
3. 分類結果を整理する

### 分類基準

| カテゴリ | 基準 | 次ラウンドでの扱い |
|---------|------|------------------|
| **Accepted** | 全員が価値を認める / 複数ロールから類似提案 | 記録して保持 |
| **Controversial** | ロール間で評価が分かれる / トレードオフがある | 次ラウンドの Phase 3 で重点議論 |
| **Frontier** | 実装方法が不明 / 革新的だが裏付けが弱い | 次ラウンドで探索を深化 |

### 重要なルール

- **合意形成しない**: Controversial は解消せず、対立構造を明記して記録する
- **Frontier を切り捨てない**: 実装不明でも可能性があれば記録する
- **各ラウンドで累積**: 前ラウンドの分類結果に今ラウンドの結果を追加する

## Phase 3: ユーザーフィードバック + 深掘り

### 目的

ユーザーが議論に参加し、チームが深掘りする。

### 手順

1. Lead が Phase 2 の分類結果をユーザーに表示する:

```
── Round {N} 分類結果 ──

**Accepted ({count}件):**
{accepted_items_summary}

**Controversial ({count}件):**
{controversial_items_with_tensions}

**Frontier ({count}件):**
{frontier_items_with_unknowns}

気になるアイデアや質問があればどうぞ！
特になければ「次のラウンド」で続行、「wrap」で終了できます。
```

2. ユーザーのフィードバックを会話ターンで受け取る（平文の問いかけ）
3. フィードバックの内容に応じてチームに深掘りを指示する:
   - 特定のアイデアへの質問 → 関連するロールに send_message で深掘りを依頼
   - 新しい方向性の提案 → 全ロールに send_message で新方向を共有
   - 対立への意見 → Controversial の該当ロールに send_message で議論を促す
4. チームからの深掘り結果を wait_agent で受け取り、論争メモリを更新する

### 深掘り時の send_message テンプレート（対象ロール宛）

```
## ユーザーフィードバック — 深掘り依頼

ユーザーから以下のフィードバックがありました:
{user_feedback}

これについて、あなたの視点から深掘りしてください:
- 追加のアイデアや変形
- リスク・機会の新たな発見
- 他のアイデアとの接続

深掘り結果の全文を出力してください。
```

## ラウンドサマリー

### 表示タイミング

各ラウンドの Phase 3 完了後（ユーザーの深掘りが終わった後）に表示する。

### フォーマット

```
── Round {N} ──
Accepted: {count}件
Controversial: {count}件（{対立の概要}）
Frontier: {count}件
Total ideas explored: {cumulative_count}
```

## ラウンド終了条件

### 次ラウンドへ進む条件

- ユーザーが「もっと」「続き」「次のラウンド」「深掘り」等と言った
- ユーザーが新しいテーマ・方向性を提示した

### セッション終了条件

- ユーザーが「wrap」「まとめて」「終わり」「十分」等と言った
- ラウンドサマリー表示後にユーザーが明示的に終了を選択した

### 終了時の表示

```
壁打ちを終了します。

── Session Summary ──
Rounds: {total_rounds}
Accepted: {count}件
Controversial: {count}件
Frontier: {count}件
Total ideas explored: {total_count}

`$team-brainstorm wrap` でアイデアをメモに整理できます。
```

## Round 2 以降の特別ルール

- **Controversial と Frontier に集中**: Round N+1 では前ラウンドの Controversial と Frontier を優先的に深掘りする
- **新しいアイデアも歓迎**: 深掘りだけでなく、新しい方向性のアイデアも出してよい
- **Accepted は安定**: 一度 Accepted になったアイデアは、新情報がない限り再議論しない
- **分類の変更あり**: 深掘りの結果、Controversial → Accepted や Frontier → Controversial への変更は起こりうる

## エラーケース

### メンバー spawn 失敗

- 成功したメンバーが 2 名以上: 残りのメンバーで発散を続行
- 成功したメンバーが 1 名以下: close_agent → セッション中断

### メンバーからの報告がない

- 報告がないメンバーは発散から除外
- 2 名以上の報告があれば続行
